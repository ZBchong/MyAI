# Video Structure

## Douyin Default

Length: 30-45s preferred.

1. Hook card, 2-3s: 痛点/结果 + 一句话承诺.
2. Pain card, 3-5s: three short pain lines.
3. Method card, 3-5s: say what AI will build or solve.
4. Proof section, 15-25s: use real screen recording, crop and zoom relevant UI, add top label and bottom explanation.
5. Result/summary, 5-8s: show outcome and why it matters.
6. CTA ending, 3-5s: use persona and comment CTA.

Default ending:

- Prefer `assets/ending/like-ending-template.mp4` when it exists.
- The final LIKE/点赞 animation from the approved sample should be appended as real video, not recreated as a static card.
- Preserve the ending template audio unless it conflicts with user-provided voice/music.
- If no ending template exists, fall back to a white ending card with: 想看完整提示词，欢迎评论区留言。

## Xiaohongshu Default

- Reuse vertical video if it works.
- Cover should be more tutorial/search friendly.
- Caption should include steps and who it is for.

## Shot Transition Guidance

- Keep typography and brand placement consistent.
- Use simple cuts between cards.
- Use screen zoom/pan inside recording segments for smoothness.
- Avoid crossfade if it causes ghosting on large text.
- Use fade only between similar live footage segments.
- Keep the previous approved style as the baseline: clean white cards, blue top accent line, compact account header, large black title, restrained gray UI cards, and real screen recording as proof.
- In demo recordings, show the full screen by default. Use smooth zoom only around actual cursor movement, click, or pause targets, then return to the full-screen view.
- Narration should be timed to cursor intent: explain the currently indicated module, stop, then continue when the next module is shown.
