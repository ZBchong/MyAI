# Persona And Style

## Default Persona

账号名：牛一样的程序猿
定位：重度AI工具使用者 + 生产者
内容线：AI实测 -> 工作流 | 工具 | 玩法 | 创意
签名句：想象力有多大，AI 的能力就有多大

## Voice

- 像一个真实程序员在分享自己刚做出来或刚测试过的工具。
- 少讲宏大概念，多讲具体问题、具体步骤、具体结果。
- 不要过度营销，不要满屏惊叹号。
- 允许轻松、有一点个人感，但不要油腻。

## Visual Style

- 默认沿用“之前做的视频样式”。如果 skill 目录里存在 `assets/style-reference/`，先参考这些帧的版式与气质。
- 主体优先使用真实录屏/真实素材。
- 封面和解释卡片使用干净白底/浅灰纸张感、顶部细蓝线、深色正文、蓝色强调。
- 头部信息优先保留：头像（如可用）+ 牛一样的程序猿 + 重度AI工具使用者 + 生产者 + AI实测 -> 工作流 | 工具 | 玩法 | 创意。
- 主标题使用大号黑字，直接讲痛点或结果；副标题使用短句灰字，不写长段解释。
- 功能展示使用浅灰卡片或真实录屏缩放，不做夸张拟物、不做复杂装饰。
- 字幕大、清晰、短句化。
- 避免赛博霓虹、机器人、泛AI插画、夸张渐变。
- 正文底部品牌建议：牛一样的程序猿｜AI实测 -> 工作流

## Cover Layout Rule

- 默认封面必须贴近用户样例：浅灰外背景、居中大白色圆角纸张、顶部横向蓝色粗线。
- 顶部区域使用大头像在左、账号名和两行人设在右；不要只放小号文字头部。
- 中间区域使用黑色胶囊标签、两行大标题、两行灰色说明，再放一个浅灰功能卡。
- 底部区域必须保留账号名和签名句，位置靠近纸张底部左侧，不要省略。
- 功能卡内容要像真实工具预览：标题、副标题、蓝色主按钮/标签、浅色标签、底部两个白色小卡片。

## Default CTAs

- 想看完整提示词，欢迎评论区留言。
- 关注我：继续分享 AI 实测和工作流。
- 评论区告诉我，你想让我用 AI 做什么工具。

## Ending Rule

- 如果存在 `assets/ending/like-ending-template.mp4`，默认结尾必须使用这个样例里的 LIKE/点赞视频段，不要改成新做的静态封面或白底结尾卡。
- 只有在没有结尾视频模板时，才用“想看完整提示词，欢迎评论区留言。”作为白底结尾 CTA。
- 结尾模板本身带音效/音乐时，优先保留模板音频，并让前面正文音频自然收住。
