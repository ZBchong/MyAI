# Persona And Style

## Default Persona

账号名：牛一样的程序猿  
定位：重度 AI 工具使用者 + 生产者  
内容线：AI 实测 -> 工作流 | 工具 | 玩法 | 创意  
签名句：想象力有多大，AI 的能力就有多大

## Voice

- 像真实程序员在分享自己刚测试过的工具。
- 少铺垫，多讲具体问题、具体操作、具体结果。
- 语速接近真人正常讲解，可以略快，但不能像赶时间。
- 开头和结尾尤其要稳，不要为了控制时长而压缩到听不清。
- 口播要贴合画面：鼠标点哪里，就讲哪里；画面切到什么模块，就解释什么模块。
- 对金融类内容，强调“信息整理、风险提示、辅助复盘”，不要说成荐股、预测、保证收益。

## Approved Visual Style

- 第5期最终交付效果是当前最新默认标准，后续只换主题内容，不换整体风格，除非用户明确要求。
- 默认使用第4期沉淀的黑底星火粒子风格。
- 背景为近黑色，配合向上运动的橙黄、粉色、少量蓝色粒子。
- 主标题使用橙黄高对比大字，可以有轻微阴影、发光和呼吸灯动效。
- 顶部可用橙到粉渐变条放置短标签，例如“AI工具实测 · 股票基金一屏看懂”。
- 封面和开头必须出现真实工具截图，截图带细描边、柔和外发光和轻微呼吸/跳动效果。
- 视频开头的顶部文字区域必须加从左往右的扫光动效，覆盖顶部标签、主标题和副标题区域；扫光要轻微、有质感，不加可见外框，不影响文字可读性。
- 扫光动效要像第5期终版：亮带从画面左侧进入，穿过标题区域后离开；视频中抽 2 秒和 5 秒帧时位置应明显不同。
- 文字呼吸动效不要加可见外框。
- 底部必须有彩色进度条，进度随时间推进。
- 字幕放在下方安全区内，位置略高于进度条，避免太靠底。

## Cover Layout Rule

- 首屏必须一眼看懂主题和痛点。
- 标题可以更抓人，但金融主题不能承诺收益或暗示确定涨跌。
- 推荐结构：
  - 顶部渐变标签：内容类型 + 主题关键词。
  - 中部大标题：痛点或强结果。
  - 副标题：澄清价值，例如“不是荐股，是把行情重点一屏看清”。
  - 真实工具截图：从录屏中截取最能证明工具效果的一帧。
  - 底部短引导：默认口播和字幕用“废话不多说，直接看演示效果吧”。
- 封面和视频第一帧保持同一套视觉结构。

## Screen Recording Style

- 真实录屏是主体证明，不要被装饰压过去。
- 先给完整画面，再根据鼠标位置短暂放大。
- 放大时保持清晰，不能把文字裁到读不清。
- 模块讲解要直切重点：它是什么、看哪里、对用户有什么用。
- 不要让字幕、口播和画面各说各的。
- “放大看这里”可以保留为短标签，但后面不要追加说明文字。
- 录屏讲解必须使用第5期终版的双层证明结构：上层原始录屏上下文 + 橙色追踪框，下层放大同一追踪区域；不能退回小截图卡片、静态缩略图或无意义的大面积空背景。
- 默认几何参考：上层区域约 `x=40,y=260,w=1000,h=560`，下层区域约 `x=40,y=930,w=1000,h=590`，底部进度条约 `y=1912,h=8`。
- 上层区域优先保证原始录屏显示完整、比例自然、上下文清楚，不为了填满框而强行裁掉重要画面。
- 下层区域优先保证橙框内容被清晰放大，允许根据实际比例保留合理暗色背景；不要求完全铺满蓝框。失败的是目标文字看不清、错误区域被放大，或小图无意义地缩在中间。

## Default CTAs

- 口播：要是觉得这个工具不错，欢迎大家评论区留言。
- 画面：感谢大家支持，多多点赞收藏加关注吧！
- 发布文案 CTA：觉得这个工具不错，欢迎评论区留言。

## Ending Rule

- 默认不再使用 LIKE 结尾视频。
- 默认加入背景音乐：用户提供背景音时优先使用用户文件；用户未提供时，默认使用 `D:\6.AI\自媒体\背景音乐_洪荒之力.MOV`。
- 背景音乐必须低音量铺底，片头淡入、片尾淡出，并在人声段自动压低，保证口播清晰。
- 结尾使用黑底星火 CTA 图卡，文案只围绕本期内容和评论互动。
- 删除与本期主内容无关的提示词、工具、项目或泛泛引导。
